## Deskripsi
Lokasi kampus memiliki hierarki bertingkat. Digunakan untuk:
- Menentukan lokasi kerusakan saat membuat laporan
- Auto-assign staff berdasarkan lokasi
- Filter dan grouping laporan

## Hierarki
```
UNIVERSITAS (root)
  └── FAKULTAS (10 fakultas)
        └── JURUSAN
              └── RUANGAN / AREA
```

## Implementasi
- Self-referencing table: `locations.parent_id → locations.id`
- Recursive CTE untuk query tree
- Endpoint `/api/locations/tree` mengembalikan nested JSON

## Endpoints
- `GET /api/locations` — flat list
- `GET /api/locations/tree` — nested tree
- `GET /api/locations/:id` — single
- `POST /api/locations` — create (admin)
- `PATCH /api/locations/:id` — update (admin)
- `DELETE /api/locations/:id` — delete (admin)

## Staff-Location Mapping (FR-29 / UC-29)
- `GET /api/staff-locations/my` — lokasi staff login
- `GET /api/staff-locations/staff/:staffId`
- `GET /api/staff-locations/location/:locationId/staff`
- `POST /api/staff-locations` — assign staff ke lokasi
- `DELETE /api/staff-locations/:staffId/:locationId` — remove
